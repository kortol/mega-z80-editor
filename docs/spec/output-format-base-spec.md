# 📘 Output Format Base Specification

*(mz80-as Output Contract – Analyze-complete based)*

---

## 1. 目的と位置づけ

Output Format Base Spec は、mz80-as が生成する **すべての成果物の共通前提条件と責務分離**を定義する内部仕様である。

```
Node[] (analyzed, resolved)
   ↓
[ emit / output ]
   ↓
.rel / .lst / .sym / (future outputs)
```

* 入力は **Analyze Phase 完了後の Node[]**
* 出力は **Analyze で確定した意味を忠実に表現する**
* 出力フェーズは **意味を解釈しない**

---

## 2. Output フェーズの前提条件（Inputs）

Output フェーズは、以下を **前提条件（Quality Gate）** とする。

1. Node ABI Spec 準拠
2. Source ABI Spec 準拠
3. Analyze Phase Spec 完了済み
4. 未解決シンボルが存在しない
5. 命令サイズ・配置アドレスが確定している

この条件を満たさない入力は **出力フェーズの契約違反**とする。

---

## 3. Output フェーズの責務（共通）

Output フェーズは以下のみを行う。

### 3.1 確定情報の直列化

* Analyze により確定した以下の情報を出力する：

  * アドレス
  * 生成バイト列
  * シンボル表
  * 元ソース対応行

---

### 3.2 表示・保存形式への変換

* 出力フォーマットごとに

  * バイナリ
  * テキスト
  * シンボル定義
    へ変換する

👉 **意味の再解釈は禁止**

---

## 4. Output フェーズの禁止事項（最重要）

Output フェーズは以下を **絶対に行ってはならない**。

❌ 式評価
❌ シンボル解決
❌ 命令サイズ計算
❌ 相対ジャンプ距離再計算
❌ Node.kind / Node.source の変更
❌ マクロ展開

---

## 5. Output 対象と共通保証

### 5.1 出力対象 Node

出力に影響するのは以下の Node のみ。

* `NodeInstr`
* `NodePseudo`（データ生成系）
* `NodeLabel`（シンボル定義）
* `NodeEmpty`（lst 行対応のみ）

---

### 5.2 行順保証

* 出力は **Node 配列順を論理行順として使用**
* マクロ展開後も **行順は保持されている**前提

---

## 6. Source の使用ルール（共通）

Output フェーズは **Source を唯一の由来情報**として使用する。

### 6.1 LST / エラー表示

* 行番号：`node.source.line`
* 表示テキスト：`node.source.text`

### 6.2 出力フォーマット間の一貫性

* `.lst` と `.sym` と `.rel` の行対応は

  * Source を基点に追跡可能であること

---

## 7. Output フォーマットの分類（Base）

| 種別    | 目的      | 例      |
| ----- | ------- | ------ |
| バイナリ系 | 実行・リンク  | `.rel` |
| 可読系   | デバッグ・確認 | `.lst` |
| 情報系   | リンク・解析  | `.sym` |

この分類は **役割分離のための論理区分**であり、
実装上の依存関係を作ってはならない。

---

## 8. フォーマット間の独立性（重要）

* 各出力フォーマットは **独立に生成可能**でなければならない
* `.lst` を生成しなくても `.rel` が生成できること
* `.sym` は `.rel` の副作用であってはならない

👉 *「一つ落ちると全部落ちる」構造を禁止*

---

## 9. エラー処理契約

* Output フェーズのエラーは

  * I/O エラー
  * 出力形式制約違反
    のみに限定される

* ソース構文・意味エラーは **Analyze 以前の責務**

---

## 10. 差分検証（Regression Testing）前提

本 Spec は以下を可能にする。

* legacy / PEG 切替時の

  * `.rel` バイナリ比較
  * `.sym` 内容比較
  * `.lst` 行対応比較
* CI における自動回帰検証

---

## 11. 将来拡張方針

* 新しい出力形式（例：JSON, ELF, MAP）は

  * 本 Base Spec に準拠した上で追加する
* Output Base Spec の変更は **破壊的変更**として扱う

---

## 12. 設計の芯（要約）

* Output は「確定した意味の写像」
* 意味を再計算しない
* Source を由来の唯一根拠とする
* フォーマット間は独立
